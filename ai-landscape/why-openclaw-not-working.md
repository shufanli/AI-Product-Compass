# 你给龙虾装了 100 个技能，它还是干不好活——问题出在哪？

OpenClaw 爆火，所有人都在装 skill。但绝大多数人的体验是一样的：装了一堆技能，让它做一件完整的事，做不起来。让它写篇推文，写出来没法看；让它做个运营方案，输出一堆正确的废话。

为什么？

---

## 一、不是技能不够，是经验为零

我们对 AI agent 有一个根本性的误解：**以为给它足够多的技能，它就能干好活**。

这就像你招了一个刚毕业的实习生，给他买了 100 本教科书，然后说："去，把增长做起来。"

他不是不会写字，不是不懂营销理论，他就是做不好。为什么？因为他没踩过坑。

Anthropic（Claude 的母公司）的增长负责人，一个人扛了一家 3800 亿美元估值公司的全部增长工作。他怎么用 AI 的？他不会让 AI 一口气写一篇完整的文章。他会：

- 让一个 agent **只写 30 个字以内的标题**
- 让另一个 agent **只写 90 个字以内的描述**
- 每个子任务都有明确的约束和验收标准

为什么他知道这么拆？因为他试过不拆，效果很烂。他试过拆成 50 个字，还是不行。他从无数次失败中知道了"30 个字"这个数字。

**这个"30"不在任何 skill 里。它是教训。**

龙虾呢？龙虾没有教训。你跟它说"写一篇推文"，它只会一口气输出一整篇。它不知道标题和正文应该分开写，不知道 30 个字是一个关键约束，不知道哪些词在这个平台上有流量，哪些没有。

**Skill 是教科书，经验是错题本。我们都在给龙虾买教科书，但真正让它考高分的是错题本。**

---

## 二、普通人用不好，不只是权限问题

很多人把 OpenClaw 用不好归因于"能控制的东西太少"——程序员能把代码仓库、服务器交给它，普通人的工具大多没给它开放权限。

这只是表面原因。

更深层的问题是：**程序员的工作天然有对错信号**。代码跑通了就是对，报错了就是错，测试通过了就是过，没通过就是没通过。这个反馈回路是自动的，agent 每一步都知道自己做得好不好。

但"写一篇好推文"有这个信号吗？没有。什么叫好？阅读量高叫好？转发多叫好？品牌调性对叫好？agent 不知道，你也很难用一句话告诉它。

所以本质上，不是"普通人的工具没开放权限"这么简单，而是**普通人的工作缺少可让机器判断的成功标准**。

程序员用得好，不是因为程序员更懂 AI，而是因为程序员的工作恰好是 AI 最容易判断对错的领域。

---

## 三、那怎么才能让龙虾真正好用？

答案不是再装 10 个 skill。

答案是：**把你脑子里的错题本翻译给它。**

具体怎么做？三步：

### 第一步：回忆你的教训

想想你过去做这件事，踩过哪些坑、总结过哪些经验。比如你做公众号运营：

- 标题超过 20 个字打开率断崖下降
- 开头三句话必须有冲突或悬念，否则跳出
- 周三晚上 8 点发比周一早上发阅读量高 40%
- 配图用实拍比用素材图点击率高
- 结尾一定要有一个明确的行动号召

这些东西不在任何运营教科书里。它们是你从数据和反馈里攒出来的。

### 第二步：把每个教训变成一个约束条件

不要说"写个好标题"。要说"标题在 18 个字以内，必须包含一个数字或一个反常识判断，禁止使用'重磅''震惊''必看'等词"。

**你对 agent 的约束越具体，它的输出就越接近你心目中的"好"。**

模糊的指令 = 模糊的结果。

### 第三步：一个教训一个 agent

不要让一个 agent 干完所有事。让每个 agent 只负责一个点，带着这个点的具体约束去执行。

- Agent A：只写标题（18字以内 + 含数字 + 禁用烂词）
- Agent B：只写开头（3句话 + 必须有冲突）
- Agent C：只写正文（800字以内 + 每200字一个小标题）
- Agent D：只选配图（实拍风格 + 和标题情绪一致）

然后你来做最终判断和组装。

**你不是在用 AI 替代自己，你是在把自己的判断力复制了很多份。**

---

## 四、这背后是一个更大的事

为什么现在的 agent 产品都不好用？因为所有人都在解决"给 agent 更多技能"的问题，没有人在解决"给 agent 注入经验"的问题。

技能和经验的区别是什么？

- **技能**：我知道怎么写一篇文章（语法、结构、逻辑）
- **经验**：我知道这篇文章写给谁、在什么场景下被阅读、哪些坑绝对不能踩、什么样的表达在这个特定人群里有效

技能是通用的，经验是私有的。

现在的 skill 生态，全在做通用能力。但**真正让 agent 从"能用"变成"好用"的，是你私有的经验**。

这也解释了为什么同一个工具，有人用得好有人用得烂。不是工具有差别，是喂给它的经验有差别。

---

## 五、最后一个思考

这件事让我想到一个反直觉的结论：

**AI 时代最值钱的不是 AI 能力本身，而是人的经验。**

以前经验锁在人的脑子里，很难规模化。现在经验可以翻译成 agent 的约束条件，被无限复制、并行执行。

一个资深运营10年踩过的坑，如果能被完整翻译成 agent 指令，就相当于瞬间复制了 100 个有10年经验的运营。

所以不要问"AI 会不会替代我"。要问：**"我脑子里有多少 AI 没有的东西？"**

如果你的工作只是技能，那确实危险。但如果你有大量从实战中来的、无法被教科书覆盖的判断力——恭喜你，你在 AI 时代反而更值钱了。

前提是，你要学会把这些判断力翻译给 AI。
