# 如何想一个产品 Feature

> 好的 Feature 不是凭空发明的，而是从真实世界的问题出发，精准定义，小而准地打向市场。

---

## 第一步：找到背后的真问题

所有 Feature 一定从真实世界的问题出发，再做延展。

**关键原则：** 不能为了形式、为了产品而做产品。

举例：桌面萌宠类产品，表面上是"提供情绪价值"，但情绪价值背后也有一个更本质的真问题。参考奢侈品的底层逻辑——"不用开口，别人就知道我的阶级"，这才是真问题。

**判断是否找到了真问题的方法：**
- 这个问题在真实用户的生活/工作中是否真实存在？
- 用户现在是如何解决这个问题的（即使用很笨的方式）？
- 如果没有你的产品，用户会感到痛苦还是只是"还好"？

---

## 第二步：用一句话定义清楚问题

真问题一定可以被一句话定义清楚。

**警惕"糅合陷阱"：** 不要把几个单独不 work 的想法强行糅合在一起。一个问题本身不 work，糅合之后依然不 work，甚至更复杂。

**一句话定义之后，还要同时考虑：**

| 维度 | 问题 |
|------|------|
| 病毒传播 | 用户会自发分享给别人吗？ |
| 商业化 | 有没有清晰的变现路径？ |
| 上线速度 | 能快速验证、快速上线吗？ |

---

## 第三步：Feature 可以很小，但要足够精准

Feature 不需要很大，一个很小的点，只要足够精准，就能打向市场。

**小而精的优势：**
- 降低开发成本，缩短验证周期
- 用户的理解成本低，更容易传播
- 聚焦一个场景，体验更极致

**反例：** 大而全的功能堆砌，往往导致每个功能都平庸，没有一个能让用户记住。
