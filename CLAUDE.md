# CLAUDE.md

本仓库是 **Product Compass** — AI 产品经理的知识罗盘。

## 仓库结构

| 目录 | 用途 |
|------|------|
| `ai-landscape/` | AI 行业认知：技术趋势、模型动态、行业格局、政策伦理 |
| `methodology/` | 方法论：PM 通用框架 + AI 产品专属方法论 |
| `product-work/` | 产品实战：需求、策略、数据、设计，0→1 全流程 |
| `case-studies/` | 案例研究：竞品拆解、成功/失败复盘 |
| `templates/` | 模板库：拿来即用的文档模板 |
| `career/` | 职业成长：技能路线、书单、面试 |

## 内容规范

- 语言：中文为主，专业术语保留英文（如 RAG、Fine-tuning、RLHF）
- 文件格式：全部使用 Markdown（`.md`）
- 文件命名：小写英文 + 短横线，如 `prompt-engineering.md`
- 每篇文档开头应有一级标题和一句话摘要
- 文档内部使用二级/三级标题组织层次，避免超过四级
- 引用外部资料时附上来源链接和日期

## 常用任务

### 内容审查（Review）
检查仓库内容时关注：
1. **格式一致性** — 标题层级、命名规范、目录结构是否统一
2. **内容质量** — 是否有空文件、占位内容、未完成的草稿
3. **时效性** — AI 领域变化快，标记超过 6 个月未更新且涉及模型/行业数据的内容
4. **交叉引用** — 不同目录间相关内容是否互相链接
5. **链接有效性** — 外部链接是否仍然可访问

### 内容补全（Gap Analysis）
对照每个目录 README 中的「建议内容」列表，识别：
- 哪些主题已经有文档
- 哪些主题缺失，优先级如何
- 输出一份简洁的内容缺口报告

### 新增内容
添加新文档时：
1. 放入对应目录
2. 遵循命名规范
3. 更新所在目录的 README（如有目录索引）
4. 检查是否需要与其他目录内容交叉引用

## 注意事项

- 不要删除或大幅改动已有内容，除非用户明确要求
- 建议性修改以 issue 或注释形式提出，让用户决定
- AI 行业信息更新频繁，写入具体模型能力/数据时标注日期
- 模板文件应保持通用性，不要填入具体项目信息
