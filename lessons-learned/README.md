# Lessons Learned — 踩过的坑

> 记录工作中犯过的错误、遇到的坑，以及从中学到的教训。
> 写下来，是为了不再重蹈覆辙。

---

## 目录结构建议

每条记录单独一个文件，命名格式：`YYYY-MM-DD-简短描述.md`

例如：
- `2026-03-26-需求没对齐就开始排期.md`
- `2026-04-01-忽略用户反馈导致功能方向偏了.md`

---

## 记录模板

```markdown
# [错误标题]

**日期：** YYYY-MM-DD  
**场景：** （发生在什么项目/情境下）

## 发生了什么

（简述事件经过）

## 根本原因

（为什么会发生？）

## 影响

（造成了什么后果？）

## 教训 & 改进

（下次怎么避免？）
```

---

*从错误中学习，是成长最快的路。*
