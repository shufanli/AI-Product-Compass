# 竞品分析工作流

一句话摘要：用"五种结构性死法"框架审视竞争格局，判断你的产品在不在死亡线上，找到真正的差异化定位。

> 来源：`case-studies/ppt-disappeared-apps-ai-opportunities.md`（移动互联网消失 App 的结构性分析）+ `case-studies/tencent-qclaw-wechat-agent.md`（腾讯 QClaw 战略拆解）

---

## 一、结构性竞争分析：五种死法框架

### 核心理念

产品不是被竞争对手打败的，是被**结构性力量**淘汰的。竞品分析的第一步不是看功能对比，而是判断你所在的赛道有没有结构性风险。

### 五种结构性死法

| 死法 | 移动时代受害者 | 一句话 | AI 时代的对应 |
|------|------------|--------|-------------|
| **系统吞噬** | 猎豹（6 亿月活→市值跌 90%） | 你的功能变成了系统的一个按钮 | Apple Intelligence / 华为小艺 / 小米系统级 AI |
| **巨头碾压** | 豌豆荚（10 亿估值→2 亿卖身） | 你帮巨头验证了需求，巨头来收割 | OpenAI 做 ChatGPT + GPTs，字节做豆包 |
| **需求消失** | WiFi 万能钥匙（9 亿用户→被遗忘） | 基础设施升级，问题不存在了 | AI API 价格一年降 90%，"能调用 AI"不再是卖点 |
| **超级 App 虹吸** | 美图（市值千亿→跌 97%） | 你的功能变成了别人的一个页面 | 用户在豆包/ChatGPT 里一句话就能解决 |
| **工具宿命** | 墨迹天气（6.5 亿用户→IPO 被否） | 干完活用户就走了 | AI 翻译翻完就走，AI 摘要摘完就走 |

## 二、产品生死线自检（5 问）

对你的产品逐一回答以下问题。任何一个答案不对，就是你的死亡线：

### 自检 Checklist

- [ ] **系统吞噬**：下个操作系统更新会自带你的功能吗？
  - 如果是 → 你做的是通用能力，会被系统吃掉。出路：做系统不会做的**垂直深度**

- [ ] **巨头碾压**：模型厂商明天做了你的功能，用户会走吗？
  - 如果是 → 你是 GPT wrapper。出路：拥有**场景和数据**，不是模型调用能力

- [ ] **需求消失**：AI 免费那天，你的产品还值钱吗？
  - 如果不值钱 → 你在卖"AI 能力"本身。出路：卖 **AI + 场景理解 = 结果**

- [ ] **超级 App 虹吸**：用户一句 prompt 能替代你吗？
  - 如果能 → 你是单功能工具。出路：嵌入生态（做 MCP / GPTs）或做一句 prompt 解决不了的**重交付**

- [ ] **工具宿命**：用户用第 100 次和第 1 次，体验有区别吗？
  - 如果没区别 → 用完即走。出路：**记忆和个性化**——越用越懂你

## 三、活下来的产品共性分析

### 核心判断

**不是在卖 AI 能力，而是在卖 AI + 场景 = 不可替代的结果。**

### 参考案例

| 产品 | 关键数据 | 为什么能活 |
|------|---------|-----------|
| Cursor | ARR 20 亿美元，估值 293 亿 | 完整开发工作流，不是单功能 |
| Midjourney | 年收入 5 亿，<100 人，从未融资 | 自研模型 + 审美壁垒 |
| Perplexity | 月活 4500 万，ARR 预计 6.56 亿 | 重新定义搜索体验 |
| Notion AI | AI 绑定率从 20%→50%+，贡献一半 ARR | AI 嵌入已有工作流 |

### 存活模式 Checklist

- [ ] 你拥有别人没有的**数据或流程知识**吗？
- [ ] 你的产品是**嵌入用户已有工作流**，还是要求用户新开一个工具？
- [ ] 用户对你的依赖是否随使用时间**递增**而非递减？
- [ ] 竞品需要多长时间才能复制你的核心价值？是代码层面还是数据/知识层面？

## 四、战略拆解方法

### 分析一个竞品/巨头动作时，问这些问题

以腾讯 QClaw 为例，展示完整的分析框架：

#### 4.1 表面事件 vs 底层逻辑

| 维度 | 分析问题 |
|------|---------|
| 时间线 | 这个动作之前发生了什么？触发条件是什么？ |
| 表面 vs 本质 | 表面看到的是什么？底层战略意图是什么？ |
| 防御 vs 进攻 | 这是"不做就会失去"还是"可以赚更多"？ |

**QClaw 案例**：表面是"腾讯做 AI Agent 工具"，本质是"防止飞书成为 AI Agent 控制台，保住微信的生产力入口"。

#### 4.2 对我的影响评估

| 维度 | 分析问题 |
|------|---------|
| 直接威胁 | 巨头这个动作会直接吃掉我的市场吗？ |
| 间接机会 | 巨头做了平台层，我能在上面做什么？ |
| 空白地带 | 巨头做了 80% 通用场景，剩下 20% 我能做什么？ |

#### 4.3 小公司的五个可行方向（平台生态思维）

| 方向 | 做什么 | 核心壁垒 |
|------|--------|---------|
| 垂直 Skill 包 | 法律/财税/医疗的专业 Agent | 行业数据 + 流程知识 |
| 私有数据连接器 | 让平台能读写企业内部系统 | 集成能力 + 行业关系 |
| 结果分发产品 | Agent 产出结果的模板化 + 分发 | 场景理解 + 分发渠道 |
| 长尾行业集成商 | 数字化程度低的行业整体方案 | 地面服务能力 |
| 垂直领域数据层 | 特定行业的结构化数据库 | 数据资产 |

## 五、竞品分析输出模板

完成分析后，输出：

### 5.1 竞争格局判断

| 维度 | 判断 |
|------|------|
| 所在赛道的结构性风险 | 哪种"死法"最可能发生？ |
| 生死线自检结果 | 5 问中有几个亮红灯？ |
| 最大威胁来源 | 系统/巨头/基础设施变化/超级 App？ |

### 5.2 竞品对比

| 维度 | 我 | 竞品 A | 竞品 B |
|------|---|--------|--------|
| 核心卖点 | ___ | ___ | ___ |
| 场景深度 | ___ | ___ | ___ |
| 数据壁垒 | ___ | ___ | ___ |
| 用户切换成本 | ___ | ___ | ___ |
| 结构性风险 | ___ | ___ | ___ |

### 5.3 差异化定位

- **我不该碰的**：___（巨头会做 / 系统会自带 / 价格在清零）
- **我应该做的**：___（有场景知识 / 有数据壁垒 / 有重交付能力）
- **底层经济逻辑**：___（成本结构、定价模型、壁垒来源）
