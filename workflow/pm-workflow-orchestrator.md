# PM 工作流编排

一句话摘要：将所有 skill 串联为完整的产品全链路，定义每个 skill 的输入、输出和触发时机。

---

## 全链路总览

```
一个想法
  │
  ▼
┌─────────────────┐
│ 1. 竞品分析       │  ← 先看别人怎么做的、结构性风险在哪
│    输出：竞争格局   │
│    判断 + 差异化    │
│    定位             │
└────────┬────────┘
         ▼
┌─────────────────┐
│ 2. 产品定义       │  ← 找真问题、一句话定义、生死线自检
│    输入：竞品分析   │
│    输出：产品定义   │
│    + 生死线自检     │
└────────┬────────┘
         ▼
┌─────────────────┐
│ 3. 产品哲学审视    │  ← 感知价值 + 启动效应 + 认知放松度
│    输入：产品定义   │
│    输出：策略优化   │
│    建议             │
└────────┬────────┘
         ▼
┌─────────────────┐
│ 4. PRD 撰写       │  ← 框架→AI 生成→交叉审查→CI Spec
│    输入：产品定义   │
│    + 策略优化建议   │
│    输出：PRD        │
└────────┬────────┘
         ▼
   ┌─────┴─────┐
   ▼           ▼
┌────────┐ ┌────────┐
│ 5. 页面 │ │ 6. 文案 │  ← 可并行执行
│ 设计审视 │ │ 定义   │
└────┬───┘ └───┬────┘
     └─────┬───┘
           ▼
┌─────────────────┐
│ 7. 上线前埋点     │  ← 漏斗定义→埋点→看板→Go/No-Go
│    和数据漏斗      │
│    输出：发布      │
│    Checklist       │
└────────┬────────┘
         ▼
      [ 上线 ]
         │
         ▼
┌─────────────────┐
│ 8. 上线后数据     │  ← T+1 对齐→漏斗分析→增长斜率→行动项
│    分析            │
│    输出：迭代      │
│    行动项          │
└────────┬────────┘
         │
         ▼
    回到 Step 4
   （下一个迭代）
```

## 各 Skill 的输入输出定义

| # | Skill | 输入 | 输出 | 触发时机 |
|---|-------|------|------|---------|
| 1 | [竞品分析](competitive-analysis.md) | 一个想法/方向 | 竞争格局判断、差异化定位、结构性风险 | 有新产品想法时 |
| 2 | [产品定义](product-definition.md) | 竞品分析输出 | 一句话定义、三维评估、MVP 边界、生死线自检 | 竞品分析完成后 |
| 3 | [产品哲学](product-philosophy.md) | 产品定义输出 | 感知价值判断、启动效应风险点、优化建议 | 产品定义完成后 |
| 4 | [PRD 撰写](prd-writing.md) | 产品定义 + 哲学审视输出 | 完整 PRD（含 AI 不确定性处理、CI Spec） | 产品定义和哲学审视完成后 |
| 5 | [页面设计审视](page-design-review.md) | PRD / 设计稿 / 线上页面 | 按优先级排列的设计问题清单 | 有设计稿或页面可审视时 |
| 6 | [文案定义](copy-definition.md) | PRD / 设计稿 | 各场景的 UX 文案 | 与页面设计审视并行 |
| 7 | [上线前埋点准备](pre-launch-data-funnel.md) | PRD + 设计稿 | 完整埋点方案、数据看板、Go/No-Go 决策 | 开发接近完成时 |
| 8 | [上线后数据分析](post-launch-data-analysis.md) | 线上数据 | 漏斗分析、增长斜率、迭代行动项 | 上线后 T+1 |

## 使用方式

### 完整链路（新产品/新功能）

当面对一个全新的产品想法时，按顺序执行 1→2→3→4→5&6→7→8。

### 部分链路（常见场景）

| 场景 | 执行哪些 Skill |
|------|---------------|
| 有了想法，想快速评估值不值得做 | 1（竞品分析）→ 2（产品定义，重点做生死线自检） |
| 已有产品，准备新功能迭代 | 4（PRD）→ 5&6（设计+文案）→ 7（埋点） |
| 功能已上线，需要数据复盘 | 8（上线后数据分析） |
| 有设计稿，需要审视质量 | 5（页面设计审视）+ 6（文案定义） |
| 准备发布，做最后检查 | 7（上线前埋点准备，重点做 Go/No-Go） |

### 迭代循环

上线后数据分析（Skill 8）的输出——迭代行动项——会回到 PRD 撰写（Skill 4）作为下一轮迭代的输入。形成闭环：

```
定义 → 设计 → 上线 → 数据 → 迭代 → 定义 → ...
```
