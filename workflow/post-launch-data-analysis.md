# 上线后数据分析工作流

一句话摘要：发布后 24 小时内完成数据对齐，逐层读数识别漏斗断裂点，用增长斜率而非绝对数判断产品健康度。

> 来源：`methodology/launch-data-funnel.md`

---

## Step 1：T+1 数据对齐会（发布后 24 小时内）

**目标：** 快速确认数据可用性，识别漏斗断裂点。

### 1.1 看板健康检查（5 min）

- [ ] 所有看板是否正常加载
- [ ] 数据是否在正常更新
- [ ] 有无明显的数据异常（如某层数据为 0）

### 1.2 漏斗通览（10 min）

- [ ] 逐层读数，标注每层转化率
- [ ] 识别转化率异常低的环节
- [ ] 与预期对比（行业 benchmark 或产品预估）

### 1.3 数据质量排查（10 min）

- [ ] 交叉验证：不同埋点间数据是否自洽
- [ ] 内部用户是否已过滤
- [ ] 是否有重复计数或遗漏

### 1.4 行动项输出（5 min）

- 数据 Bug → 研发修复
- 漏斗瓶颈 → 产品优化
- 缺失埋点 → 补埋排期

| F6 | 转化成功 | ___ | ___% | F6/F5 |

**端到端转化率：** F6/F1 = ___%

### 关键转化率参考值

| 指标 | 参考范围 | 说明 |
|------|---------|------|
| 消息曝光→点击 | 3%-10% | 飞书/社群消息；年度报告类可达 8%-10% |
| 落地页→核心行为 | 20%-40% | 取决于产品价值感知和操作成本 |
| 核心行为→转化页 | 30%-60% | 完成核心体验后的自然流转 |
| 转化页→付费意向 | 5%-15% | 点击付费按钮 / 填写信息 |
| 分享率 | 5%-20% | 取决于分享激励设计 |
| 裂变系数（K-factor） | >1 为理想 | 每个用户带来 >1 个新用户 |

**裂变系数 K = 分享率 × 每次分享带来的新用户数 = ___**
- K > 1 = 自发增长 ✅
- K < 1 = 需外部驱动 ⚠️
