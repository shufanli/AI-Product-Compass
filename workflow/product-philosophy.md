# 产品哲学工作流

一句话摘要：在设计产品之前，用感知价值理论和启动效应原理审视产品策略，确保用户"感知到"的价值与你设计的价值一致。

> 来源：`methodology/perceived-value-theory.md` + `methodology/priming-and-product-design.md`

---

## 一、感知价值审视

### 核心公式

```
感知价值 = 感知收益 / 感知付出
```

- **感知收益** = 功能 + 情感 + 社会认同 + 品牌联想
- **感知付出** = 价格 + 时间成本 + 购买风险 + 认知负担

### 两条提升路径

| 路径 | 策略 | 举例 |
|------|------|------|
| A：提升分子（增加收益感知） | 改包装、提品牌形象、增加情感共鸣 | 优化视觉触感、广告叙事、社群认同 |
| B：降低分母（降低付出感知） | 分期免息、锚定效应、简化购买流程 | "每天只需一杯咖啡的钱"、减少步骤 |

### 五条产品启示

1. **功能 ≠ 价值**：用户感知到的价值才算数，再好的功能如果没有被感知到，等于不存在
2. **定价即信号**：过低的定价可能反向破坏感知价值
3. **首屏/首次体验决定感知基调**：用户的第一印象会锚定后续所有判断
4. **降低认知付出**：简洁的界面和流程本身就是在提升感知价值
5. **外部线索设计是产品力的一部分**：文案、视觉、动效、交互反馈都是感知管理

### 感知价值 Checklist

- [ ] 产品的核心功能，用户能在首次使用中感知到吗？
- [ ] 定价策略是否传递了正确的质量信号？
- [ ] 首屏/首次体验是否建立了正面的感知基调？
- [ ] 用户完成核心操作的认知负担是否足够低？
- [ ] 外部线索（文案、视觉、动效）是否在强化感知收益？

## 二、启动效应审视

### 核心机制

用户的决策很少是理性的，更多是被当下的联想状态驱动的。产品设计的本质，是管理用户的联想网络。

### 六个产品设计检查点

#### 1. 第一接触点决定后续感知

- [ ] 不同渠道来源的用户，首屏内容是否有差异化？
- [ ] 负面口碑带来的用户，第一句话是否先"消毒"而不是直接推销？
- [ ] Onboarding 的核心问题是"用户带着什么心理状态进来"，而不是"怎么展示功能"

#### 2. 品牌设计是否占领了正确的联想节点

- [ ] 产品名、颜色、字体、slogan 激活的联想是否与目标一致？
- [ ] 产品是否与某个正向场景绑定得足够紧？

#### 3. 价格展示时机是否合理

- [ ] 是否让用户先体验价值，再看价格？
- [ ] 越早展示价格，用户越早进入"斤斤计较"模式——这个时机对吗？

#### 4. 社会监督感是否被利用

- [ ] 公开进度条、好友动态、排行榜等设计是否合理运用了"有人在看你"的感知？

#### 5. 负面体验后是否有"清洗"机会

- [ ] 负面体验（bug、客服差、支付失败）后，是否有真诚道歉、退款、超预期补偿？
- [ ] 危机处理的本质是认知重置，不只是情感安抚

#### 6. 是否引导用户"身体参与"

- [ ] Onboarding 是否引导用户做第一个动作，而不是给他看一堆介绍？
- [ ] 用户完成一个小操作后，对产品的认同感会比"只看一眼"高得多

## 三、认知放松度审视

### 核心机制

处理流畅 = 大脑判定"安全/真实/可信"。

- **放松状态**：用户心情好、相信直觉、降低警惕、更容易接受产品
- **紧张状态**：用户警觉、多疑、分析性增强、更容易流失

### Checklist

- [ ] UI 的字体、对比度、排版是否让用户处于认知放松状态？
- [ ] 文案是否简洁、朗朗上口？
- [ ] 品牌名、产品功能命名是否好记好发音？
- [ ] 新功能是否通过多次轻量级曝光（角标、空态提示）建立熟悉感，而非一次重量级弹窗？
- [ ] 在需要用户深思熟虑的决策节点（大额付款、隐私授权），是否适当增加了摩擦？

---

## 产出物

完成以上审视后，应输出：

1. **感知价值判断**——当前产品的感知收益和感知付出各是什么，比值是否健康
2. **启动效应风险点**——哪些接触点的联想管理有问题
3. **认知放松度评估**——产品整体是让用户放松还是紧张
4. **优化行动项**——按优先级排列的具体改进建议
